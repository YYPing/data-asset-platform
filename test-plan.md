# 数据资产平台 - 全面测试计划

## 测试目标
对照设计文档，验证数据资产平台的核心功能是否符合设计要求。

## 测试范围
1. 客户管理模块
2. 项目管理模块  
3. 系统登记模块
4. 价值评估模块
5. 用户认证模块

## 测试环境
- API服务: http://localhost:3000
- 测试工具: Node.js HTTP客户端
- 测试数据: 根据设计文档生成

## 测试用例设计

### 1. 用户认证测试
#### 1.1 登录功能
- 正常登录（admin/admin123）
- 错误密码登录
- 不存在的用户登录
- 获取当前用户信息
- Token验证

#### 1.2 权限验证
- 未授权访问API
- Token过期验证
- 非法Token验证

### 2. 客户管理测试
#### 2.1 客户编码生成
- 验证编码格式：C+年月+4位序号
- 验证编码唯一性
- 验证序号递增

#### 2.2 客户CRUD操作
- 创建客户（必填字段验证）
- 查询客户列表（分页、筛选）
- 获取客户详情
- 更新客户信息
- 删除客户（逻辑删除）
- 客户状态切换

#### 2.3 业务规则验证
- 公司名称唯一性校验
- 营业执照号唯一性校验
- 手机号格式验证
- 状态管理验证

### 3. 项目管理测试
#### 3.1 项目编码生成
- 验证编码格式：P+年月+4位序号
- 验证编码唯一性

#### 3.2 项目CRUD操作
- 创建项目（关联客户）
- 查询项目列表
- 获取项目详情
- 更新项目信息
- 项目进度更新
- 项目阶段更新

#### 3.3 业务规则验证
- 客户存在性验证
- 项目阶段有效性验证
- 进度值范围验证（0-100%）

### 4. 系统登记测试
#### 4.1 系统登记
- 创建系统信息
- 系统信息完整性验证
- 关联项目验证

#### 4.2 系统查询
- 按项目查询系统列表
- 系统状态筛选
- 是否选中筛选

### 5. 价值评估测试
#### 5.1 评估算法验证
- 业务价值评分计算
- 数据质量评分计算  
- 合规风险评分计算
- 技术可行性评分计算
- 综合评分计算（加权平均）

#### 5.2 评估建议生成
- 建议等级划分（立即挖掘/优先挖掘/评估后决定/暂不挖掘）
- 优先级设置（high/medium/low/none）
- 建议数据产品生成

#### 5.3 评估结果验证
- 评估结果存储
- 评估详情记录
- 审核流程验证

## 预期结果

### 成功标准
1. 所有API端点返回正确的HTTP状态码
2. 业务逻辑符合设计文档要求
3. 数据验证规则正确执行
4. 错误处理机制正常工作
5. 性能满足要求（响应时间<2秒）

### 失败处理
1. 记录失败的测试用例
2. 分析失败原因
3. 生成测试报告
4. 提出改进建议
