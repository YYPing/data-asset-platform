## 关键问题分析

### P0-1: 模型导入错误（阻塞）

**严重程度：** 🔴 严重
**影响范围：** 4 个测试文件无法加载
**阻塞测试：** 13+ 个测试用例

**问题描述：**
测试文件期望导入的模型类名与实际定义不匹配。

**具体错误：**
```python
# 测试文件期望：
from app.models.asset import Asset, AssetVersion, AssetMaterial, AssetCertificate

# 实际模型定义：
class DataAsset(Base, TimestampMixin, SoftDeleteMixin):
    __tablename__ = "data_assets"
```

**受影响文件：**
1. `tests/test_db.py` - 数据库测试
2. `tests/test_client.py` - 客户端测试
3. `tests/test_auth_comprehensive.py` - 综合认证测试
4. `tests/test_materials_comprehensive.py` - 综合材料测试

**根本原因：**
- 模型重构后类名从 `Asset` 改为 `DataAsset`
- 测试文件未同步更新
- 缺少模型导出别名

**修复建议：**
```python
# 方案 1：在 app/models/asset.py 中添加别名
Asset = DataAsset  # 向后兼容

# 方案 2：更新所有测试文件的导入语句
from app.models.asset import DataAsset as Asset
```

**优先级：** P0 - 必须立即修复
**预计修复时间：** 15-30 分钟

### P0-2: 表重复定义错误（阻塞）

**严重程度：** 🔴 严重
**影响范围：** 1 个测试文件无法加载
**阻塞测试：** 材料管理模块所有测试

**问题描述：**
SQLAlchemy 检测到 `materials` 表被重复定义。

**具体错误：**
```
sqlalchemy.exc.InvalidRequestError: Table 'materials' is already defined for this MetaData instance.
Specify 'extend_existing=True' to redefine options and columns on an existing Table object.
```

**位置：** `app/models/material.py:14`

**根本原因：**
- 可能存在多个 Material 模型定义
- 模型被多次导入
- 缺少 `extend_existing=True` 配置

**修复建议：**
```python
# 在 Material 模型中添加：
class Material(Base):
    __tablename__ = "materials"
    __table_args__ = (
        {'extend_existing': True}  # 添加此配置
    )
```

**优先级：** P0 - 必须立即修复
**预计修复时间：** 10-15 分钟

### P0-4: 异步事件循环错误（阻塞）

**严重程度：** 🔴 严重
**影响范围：** 所有异步测试
**阻塞测试：** 28 个测试错误

**问题描述：**
测试执行后事件循环被关闭，导致后续测试无法运行。

**具体错误：**
```
RuntimeError: Event loop is closed
```

**根本原因：**
- pytest-asyncio 配置不正确
- 测试 fixture 未正确管理事件循环
- 异步资源未正确清理

**修复建议：**
```python
# 在 tests/conftest.py 中添加：
import pytest

@pytest.fixture(scope="function")
def event_loop():
    """创建新的事件循环用于每个测试"""
    import asyncio
    loop = asyncio.new_event_loop()
    yield loop
    loop.close()
```

或在 `pytest.ini` 中配置：
```ini
[pytest]
asyncio_mode = auto
asyncio_default_fixture_loop_scope = function
```

**优先级：** P0 - 必须立即修复
**预计修复时间：** 30-45 分钟
