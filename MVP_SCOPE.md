### P0-3: 本地开发环境设置 ⚡ 关键
**负责人：** 后端工程师
**预计工作量：** 1-2 小时
**阻塞：** 无法运行应用程序

**决策要求：** Docker vs 本地服务

**选项 A：Docker 设置（推荐）**
- 安装 Docker Desktop for macOS
- 运行：`docker-compose up -d`
- 优点：匹配生产环境，包含所有服务
- 缺点：需要安装 Docker

**选项 B：本地服务**
- 本地安装 PostgreSQL（通过 Homebrew）
- 本地安装 Redis（通过 Homebrew）
- 跳过 MinIO（设置 `MINIO_ENABLED=false`）
- 优点：更轻量，启动更快
- 缺点：需要手动配置

**MVP 决策：选项 B（本地服务）**
- 在 48 小时窗口内设置更快
- MVP 可以禁用 MinIO
- 文件上传可以暂时使用本地文件系统

**任务：**
1. 安装 PostgreSQL：`brew install postgresql@15`
2. 安装 Redis：`brew install redis`
3. 启动服务：
   ```bash
   brew services start postgresql@15
   brew services start redis
   ```
4. 创建数据库：
   ```bash
   createdb data_asset_platform
   ```

**验证：**
```bash
psql -d data_asset_platform -c "SELECT 1;"  # 应返回 1
redis-cli ping  # 应返回 PONG
```

## 风险缓解

### 风险 1：Docker 安装问题
**概率：** 中
**影响：** 高
**缓解：** 改用本地 PostgreSQL/Redis（选项 B）
**后备方案：** 在绝对紧急情况下保留 SQLite，但功能会减少

### 风险 2：TypeScript 错误比预期更复杂
**概率：** 中
**影响：** 中
**缓解：** 前端工程师预算了 4-5 小时，可延长至 6 小时
**后备方案：** 对非关键错误使用 `@ts-ignore` 以解除构建阻塞

### 风险 3：P0-1 修复后测试套件有更多问题
**概率：** 高
**影响：** 中
**缓解：** QA + 后端配对进行测试修复，预算 2 小时
**后备方案：** 如果问题出在非关键功能中，接受 70% 通过率

### 风险 4：数据库迁移问题
**概率：** 低
**影响：** 中
**缓解：** MVP 使用新数据库，无需迁移
**后备方案：** 如果 PostgreSQL 设置失败，使用 SQLite

### 风险 5：前后端集成问题
**概率：** 中
**影响：** 中
**缓解：** 为集成测试预算 1 小时
**后备方案：** 首先修复 CORS、认证问题；延期高级功能
