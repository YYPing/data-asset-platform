## 一、蜂群开发架构

### 1.1 角色定义

蜂群开发不是简单的"分任务给多个Agent"，而是模拟一个真实的开发团队，每个Agent有明确的**角色、职责、输入输出规范**。

```
┌─────────────────────────────────────────────────────┐
│                   👑 贾维斯（我）                      │
│              项目经理 / 蜂群调度中心                    │
│  - 任务分解与分配                                     │
│  - 进度跟踪与质量把控                                  │
│  - 冲突协调与决策                                     │
│  - 黑板管理（汇总各Agent产出）                         │
└──────────┬──────────┬──────────┬──────────┬──────────┘
           │          │          │          │
    ┌──────▼──┐ ┌─────▼───┐ ┌───▼─────┐ ┌──▼──────┐
    │🏗️ 架构师 │ │💻 后端Dev│ │🎨 前端Dev│ │🧪 测试员 │
    │ (Opus)  │ │(Sonnet) │ │(Sonnet) │ │(Sonnet) │
    │         │ │         │ │         │ │         │
    │技术方案  │ │API+业务  │ │页面+组件 │ │测试用例  │
    │DB设计   │ │逻辑实现  │ │大屏展示  │ │质量验收  │
    │接口规范  │ │数据层    │ │交互逻辑  │ │Bug报告  │
    └─────────┘ └─────────┘ └─────────┘ └─────────┘
```

### 1.2 蜂群精髓在开发中的体现

| 蜂群原则 | 开发中的实现 |
|---------|------------|
| **结构化黑板** | 每个Agent产出写入约定目录，用JSON清单索引 |
| **Worker内循环** | Agent内部自我调试迭代，不消耗调度成本 |
| **Qwen守门** | 贾维斯（我）充当守门员：分配任务+验收结果 |
| **并行执行** | 无依赖的模块同时spawn多个Agent开发 |
| **成本控制** | 架构决策用Opus，编码用Sonnet，简单任务用Haiku |
| **摘要上板** | Agent完成后只汇报摘要，全文存文件 |

## 三、蜂群调度策略

### 3.1 任务分配原则

```
任务复杂度判断：
├─ 架构决策/设计类 → Opus（贵但准确）
│   例：工作流引擎设计、数据库模型设计
├─ 编码实现类 → Sonnet（性价比最优）
│   例：API开发、页面开发、测试编写
├─ 简单/重复类 → Haiku（快速便宜）
│   例：CRUD模板生成、配置文件、文档
└─ 大量代码生成 → DeepSeek（无限量）
    例：测试数据生成脚本、迁移脚本
```

### 3.2 并行度控制

```
最大并行Agent数: 4（避免上下文爆炸）
每批次等待全部完成后再启动下一批
每个Agent超时: 10分钟（复杂任务15分钟）
```

### 3.3 质量保证流程

```
Agent完成任务
    ↓
贾维斯验收（检查产出文件）
    ├─ 通过 → 合并到主代码
    ├─ 小问题 → 直接修复（不重新spawn）
    └─ 大问题 → 重新spawn修正Agent
```

### 3.4 黑板管理

每个Agent完成后，贾维斯更新 `board/tasks.json`：

```json
{
  "phase": 1,
  "batch": 2,
  "tasks": [
    {
      "id": "asset-crud",
      "agent": "Asset-Dev",
      "model": "sonnet",
      "status": "completed",
      "files": ["src/backend/app/api/v1/assets.py", "src/backend/app/services/asset.py"],
      "quality": "passed",
      "notes": "zhparser全文搜索已集成"
    }
  ]
}
```
