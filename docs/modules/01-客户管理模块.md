# 模块设计 - 客户管理模块

## 1. 模块概述

### 1.1 功能定位
客户管理模块是整个平台的入口，管理所有客户的基本信息、联系方式、合同信息等，为后续的项目开展提供基础数据支撑。

### 1.2 涉及角色
- **系统管理员**: 管理所有客户信息
- **客户经理**: 管理自己负责的客户
- **项目经理**: 查看相关客户信息

## 2. 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 客户列表查询 | P0 | 支持多条件筛选和分页 |
| 客户新增 | P0 | 录入新客户基本信息 |
| 客户编辑 | P0 | 修改客户信息 |
| 客户详情 | P0 | 查看客户完整信息 |
| 客户停用/启用 | P1 | 控制客户状态 |
| 客户导入/导出 | P1 | 批量操作 |
| 客户统计看板 | P2 | 客户数据可视化 |

## 5. 核心业务逻辑

### 5.1 客户编码生成规则
```java
@Service
public class CustomerCodeGenerator {
    
    /**
     * 生成客户编码: C + 年月 + 4位序号
     * 示例: C2024020001
     */
    public String generateCode() {
        String prefix = "C" + LocalDate.now().format(DateTimeFormatter.ofPattern("yyyyMM"));
        
        // 查询当月最大序号
        String maxCode = customerMapper.selectMaxCodeByPrefix(prefix);
        int sequence = 1;
        if (maxCode != null) {
            String seqStr = maxCode.substring(prefix.length());
            sequence = Integer.parseInt(seqStr) + 1;
        }
        
        return prefix + String.format("%04d", sequence);
    }
}
```

### 5.2 客户唯一性校验
```java
@Service
public class CustomerService {
    
    @Transactional
    public CustomerVO createCustomer(CustomerDTO dto) {
        // 1. 校验公司名称唯一性
        if (customerMapper.existsByCompanyName(dto.getCompanyName())) {
            throw new BusinessException("公司名称已存在");
        }
        
        // 2. 校验营业执照号唯一性
        if (StringUtils.isNotBlank(dto.getBusinessLicense()) && 
            customerMapper.existsByBusinessLicense(dto.getBusinessLicense())) {
            throw new BusinessException("营业执照号已存在");
        }
        
        // 3. 生成客户编码
        String customerCode = codeGenerator.generateCode();
        
        // 4. 保存客户信息
        Customer customer = new Customer();
        BeanUtils.copyProperties(dto, customer);
        customer.setCustomerCode(customerCode);
        customer.setStatus(1);
        customerMapper.insert(customer);
        
        return convertToVO(customer);
    }
}
```

## 8. 注意事项

1. **数据隔离**: 客户经理只能看到自己和公共客户
2. **编码规则**: 客户编码一旦生成不可修改
3. **关联检查**: 删除客户前检查是否有关联项目
4. **审计日志**: 所有操作记录审计日志
