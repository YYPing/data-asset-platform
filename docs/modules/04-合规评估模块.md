# 模块设计 - 合规评估模块

## 1. 模块概述

### 1.1 功能定位
合规评估模块对数据产品和数据处理活动进行全面的法规符合性检查，识别合规风险，生成整改建议，并支持律师意见和内部制度材料的管理。

### 1.2 覆盖法规
- 网络安全法
- 数据安全法
- 个人信息保护法
- 数据分类分级指南
- 等保2.0

## 2. 功能清单

| 功能点 | 优先级 | 说明 |
|--------|--------|------|
| 法规检查清单 | P0 | 自动生成检查项 |
| 合规自评估 | P0 | 逐项检查合规性 |
| 风险评估 | P0 | 识别和评估风险 |
| 律师意见管理 | P1 | 上传和管理法律意见 |
| 制度材料管理 | P1 | 内部制度文档管理 |
| 整改建议 | P1 | 生成整改方案 |
| 合规报告 | P1 | 生成合规评估报告 |

## 4. 合规检查引擎

### 4.1 检查规则定义
```java
@Component
public class ComplianceRuleEngine {
    
    /**
     * 网络安全法检查规则
     */
    public List<ComplianceCheckItem> checkCybersecurityLaw(
            DataProduct product, 
            CustomerSystem system) {
        
        List<ComplianceCheckItem> items = new ArrayList<>();
        
        // 检查项1: 数据分类分级
        items.add(checkDataClassification(product, system));
        
        // 检查项2: 重要数据保护
        items.add(checkImportantDataProtection(product, system));
        
        // 检查项3: 数据安全事件应急预案
        items.add(checkIncidentResponsePlan(system));
        
        // 检查项4: 网络安全等级保护
        items.add(checkSecurityLevelProtection(system));
        
        return items;
    }
    
    /**
     * 个人信息保护法检查
     */
    public List<ComplianceCheckItem> checkPersonalInfoLaw(DataProduct product) {
        List<ComplianceCheckItem> items = new ArrayList<>();
        
        // 检查项1: 告知同意
        items.add(checkInformedConsent(product));
        
        // 检查项2: 处理合法性
        items.add(checkProcessingLegitimacy(product));
        
        // 检查项3: 敏感个人信息处理
        items.add(checkSensitiveInfoProcessing(product));
        
        // 检查项4: 数据主体权利保障
        items.add(checkDataSubjectRights(product));
        
        // 检查项5: 跨境传输合规
        items.add(checkCrossBorderTransfer(product));
        
        return items;
    }
    
    private ComplianceCheckItem checkDataClassification(
            DataProduct product, CustomerSystem system) {
        
        ComplianceCheckItem item = new ComplianceCheckItem();
        item.setRegulationType("网络安全法");
        item.setCheckCategory("数据安全管理");
        item.setCheckItem("是否建立数据分类分级保护制度");
        item.setRequirementDescription(
            "根据数据在经济社会发展中的重要程度，以及一旦遭到篡改、破坏、泄露或者非法获取、" +
            "非法利用，对国家安全、公共利益或者个人、组织合法权益造成的危害程度，对数据实行分类分级保护"
        );
        
        // 评估
        boolean hasClassification = system.getDataClassification() != null;
        boolean hasSecurityLevel = system.getSecurityLevel() != null;
        
        if (hasClassification && hasSecurityLevel) {
            item.setCheckResult("compliant");
            item.setEvidence("已建立数据分级分类制度，数据分级为" + 
                           system.getDataClassification() + 
                           "，等保级别为" + system.getSecurityLevel());
            item.setRiskLevel("low");
        } else {
            item.setCheckResult("non_compliant");
            item.setFindings("未建立完整的数据分类分级保护制度");
            item.setRecommendations(
                "1. 制定数据分类分级管理制度\n" +
                "2. 明确数据分级分类标准\n" +
                "3. 对现有数据进行分类分级标注"
            );
            item.setRiskLevel("high");
        }
        
        return item;
    }
    
    private ComplianceCheckItem checkSensitiveInfoProcessing(DataProduct product) {
        ComplianceCheckItem item = new ComplianceCheckItem();
        item.setRegulationType("个人信息保护法");
        item.setCheckCategory("敏感个人信息");
        item.setCheckItem("敏感个人信息的处理是否符合法律规定");
        
        // 检查产品中是否包含敏感个人信息
        boolean hasSensitiveInfo = product.getOutputFields().stream()
            .anyMatch(field -> isSensitivePersonalInfo(field));
        
        if (hasSensitiveInfo) {
            // 检查是否有单独同意
            boolean hasSeparateConsent = checkSeparateConsent(product);
            // 检查是否有特定目的
            boolean hasSpecificPurpose = checkSpecificPurpose(product);
            // 检查是否进行影响评估
            boolean hasImpactAssessment = checkImpactAssessment(product);
            
            if (hasSeparateConsent && hasSpecificPurpose && hasImpactAssessment) {
                item.setCheckResult("compliant");
                item.setEvidence("敏感个人信息处理符合法律要求");
                item.setRiskLevel("medium");
            } else {
                item.setCheckResult("partial");
                List<String> gaps = new ArrayList<>();
                if (!hasSeparateConsent) gaps.add("缺少单独同意");
                if (!hasSpecificPurpose) gaps.add("处理目的不够明确");
                if (!hasImpactAssessment) gaps.add("未进行个人信息保护影响评估");
                
                item.setFindings(String.join("；", gaps));
                item.setRecommendations(
                    "1. 获取数据主体的单独同意\n" +
                    "2. 明确敏感个人信息的处理目的\n" +
                    "3. 开展个人信息保护影响评估\n" +
                    "4. 考虑对敏感信息进行脱敏处理"
                );
                item.setRiskLevel("high");
            }
        } else {
            item.setCheckResult("na");
            item.setEvidence("产品中不包含敏感个人信息");
            item.setRiskLevel("low");
        }
        
        return item;
    }
}
```

### 4.2 风险评估算法
```java
@Service
public class ComplianceRiskCalculator {
    
    /**
     * 计算整体合规风险
     */
    public RiskAssessment calculateOverallRisk(
            List<ComplianceCheckItem> checkItems) {
        
        RiskAssessment assessment = new RiskAssessment();
        
        // 按风险等级统计
        int criticalCount = 0;
        int highCount = 0;
        int mediumCount = 0;
        int lowCount = 0;
        
        for (ComplianceCheckItem item : checkItems) {
            if ("non_compliant".equals(item.getCheckResult())) {
                switch (item.getRiskLevel()) {
                    case "critical": criticalCount++; break;
                    case "high": highCount++; break;
                    case "medium": mediumCount++; break;
                    case "low": lowCount++; break;
                }
            }
        }
        
        // 计算风险评分（0-100，越高风险越大）
        double riskScore = criticalCount * 25 + highCount * 15 + 
                          mediumCount * 5 + lowCount * 1;
        riskScore = Math.min(riskScore, 100);
        
        assessment.setRiskScore(riskScore);
        
        // 确定风险等级
        if (riskScore >= 60 || criticalCount > 0) {
            assessment.setRiskLevel("high");
            assessment.setRiskDescription("高风险，需要立即整改");
        } else if (riskScore >= 30 || highCount > 2) {
            assessment.setRiskLevel("medium");
            assessment.setRiskDescription("中等风险，需要制定整改计划");
        } else {
            assessment.setRiskLevel("low");
            assessment.setRiskDescription("低风险，持续监控即可");
        }
        
        assessment.setCriticalIssues(criticalCount);
        assessment.setHighIssues(highCount);
        assessment.setMediumIssues(mediumCount);
        assessment.setLowIssues(lowCount);
        
        return assessment;
    }
}
```

## 5. API接口

### 5.1 创建合规评估
```yaml
接口: POST /api/compliance-assessments
请求体:
  projectId: long, required
  dataProductId: long
响应:
  code: 200
  data:
    assessmentId: long
    checkItems: list
```

### 5.2 执行合规检查
```yaml
接口: POST /api/compliance-assessments/{id}/execute
响应:
  code: 200
  data:
    assessmentId: long
    checkResults: list
    overallStatus: string
    riskLevel: string
```

### 5.3 更新检查项
```yaml
接口: PUT /api/compliance-assessments/{id}/check-items/{itemId}
请求体:
  checkResult: string
  evidence: string
  findings: string
响应:
  code: 200
```

### 5.4 上传律师意见
```yaml
接口: POST /api/compliance-assessments/{id}/legal-opinion

参数:
  file: File
  lawyerName: string
  lawyerFirm: string
  opinionDate: date
  summary: string

  code: 200
```
