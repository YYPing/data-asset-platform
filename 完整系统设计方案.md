# 数据资产全流程管理平台 - 完整系统设计方案

## 项目概述

### 1.1 项目背景
为支持政府数据资产登记试点方案，建设独立的数据资产全流程管理平台，实现数据资产从登记、评估、确权到运营的全生命周期管理。

### 1.2 项目目标
- 支持数据持有方管理自己的数据资产
- 支持登记中心监控全市数据资产情况
- 实现全流程材料保存和管理
- 提供大屏展示功能
- 确保系统安全可靠

### 1.3 项目范围
- 数据资产管理
- 材料上传和管理
- 合规评估管理
- 价值评估管理
- 大屏展示系统
- 系统管理和运维

### 2.2 技术栈选型

#### 前端技术栈
- 核心框架: Vue.js 3.x + TypeScript
- UI组件库: Element Plus
- 状态管理: Pinia
- 路由管理: Vue Router 4.x
- HTTP客户端: Axios
- 图表库: ECharts 5.x
- 大屏框架: DataV
- 地图组件: 高德地图 API
- 移动端: Uni-app
- 构建工具: Vite

#### 后端技术栈
- 核心框架: Python 3.10+ FastAPI
- ORM框架: SQLAlchemy 2.0
- 数据验证: Pydantic
- 异步任务: Celery + Redis
- API文档: OpenAPI (Swagger)
- 认证授权: JWT + OAuth2
- 日志框架: Loguru
- 测试框架: Pytest

#### 数据存储
- 关系数据库: PostgreSQL 14+
- 对象存储: MinIO
- 缓存: Redis 7.x
- 全文搜索: Elasticsearch 8.x
- 消息队列: RabbitMQ

#### 部署运维
- 容器化: Docker + Docker Compose
- CI/CD: GitLab CI / GitHub Actions
- 监控: Prometheus + Grafana
- 日志: ELK Stack
- 反向代理: Nginx

## 数据库设计

### 4.1 核心数据表

#### 用户表 (users)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    real_name VARCHAR(100),
    email VARCHAR(100),
    phone VARCHAR(20),
    role VARCHAR(20) NOT NULL,
    organization_id INTEGER,
    status VARCHAR(20) DEFAULT 'active',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);
```

#### 组织机构表 (organizations)
```sql
CREATE TABLE organizations (
    id SERIAL PRIMARY KEY,
    name VARCHAR(200) NOT NULL,
    code VARCHAR(50) UNIQUE,
    type VARCHAR(50),
    parent_id INTEGER,
    level INTEGER,
    contact_person VARCHAR(100),
    contact_phone VARCHAR(20),
    address TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 数据资产表 (data_assets)
```sql
CREATE TABLE data_assets (
    id SERIAL PRIMARY KEY,
    asset_code VARCHAR(100) UNIQUE NOT NULL,
    asset_name VARCHAR(200) NOT NULL,
    organization_id INTEGER NOT NULL,
    category VARCHAR(50),
    description TEXT,
    data_source TEXT,
    data_volume BIGINT,
    data_format VARCHAR(50),
    update_frequency VARCHAR(50),
    status VARCHAR(20) DEFAULT 'draft',
    created_by INTEGER,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 材料管理表 (materials)
```sql
CREATE TABLE materials (
    id SERIAL PRIMARY KEY,
    asset_id INTEGER NOT NULL,
    material_name VARCHAR(200) NOT NULL,
    material_type VARCHAR(50),
    stage VARCHAR(50) NOT NULL,
    file_path VARCHAR(500),
    file_size BIGINT,
    file_format VARCHAR(20),
    version VARCHAR(20) DEFAULT '1.0.0',
    is_required BOOLEAN DEFAULT false,
    status VARCHAR(20) DEFAULT 'pending',
    uploaded_by INTEGER,
    uploaded_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    reviewed_by INTEGER,
    reviewed_at TIMESTAMP,
    review_comment TEXT
);
```

### 4.2 索引设计
```sql
-- 用户表索引
CREATE INDEX idx_users_username ON users(username);
CREATE INDEX idx_users_role ON users(role);
CREATE INDEX idx_users_organization ON users(organization_id);

-- 数据资产表索引
CREATE INDEX idx_assets_code ON data_assets(asset_code);
CREATE INDEX idx_assets_organization ON data_assets(organization_id);
CREATE INDEX idx_assets_status ON data_assets(status);

-- 材料表索引
CREATE INDEX idx_materials_asset ON materials(asset_id);
CREATE INDEX idx_materials_stage ON materials(stage);
CREATE INDEX idx_materials_status ON materials(status);
```

## API接口设计

### 5.1 认证授权接口

#### 用户登录
```
POST /api/v1/auth/login
Request:
{
  "username": "string",
  "password": "string"
}
Response:
{
  "access_token": "string",
  "token_type": "bearer",
  "expires_in": 3600,
  "user": {...}
}
```

#### 刷新令牌
```
POST /api/v1/auth/refresh
Headers: Authorization: Bearer <token>
Response:
{
  "access_token": "string",
  "expires_in": 3600
}
```

### 5.2 数据资产接口

#### 创建数据资产
```
POST /api/v1/assets
Headers: Authorization: Bearer <token>
Request:
{
  "asset_name": "string",
  "category": "string",
  "description": "string"
}
Response:
{
  "id": 1,
  "asset_code": "DA20260208001",
  "status": "draft"
}
```

#### 查询数据资产列表
```
GET /api/v1/assets?page=1&size=20&status=registered
Headers: Authorization: Bearer <token>
Response:
{
  "total": 100,
  "page": 1,
  "size": 20,
  "items": [...]
}
```

### 5.3 材料管理接口

#### 上传材料
```
POST /api/v1/materials/upload
Headers: 
  Authorization: Bearer <token>
  Content-Type: multipart/form-data
Request:
{
  "asset_id": 1,
  "material_name": "string",
  "stage": "registration",
  "file": <binary>
}
Response:
{
  "id": 1,
  "material_name": "string",
  "file_path": "string",
  "status": "pending"
}
```
